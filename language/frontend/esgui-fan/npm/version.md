# version

* 概述
* ~
* ^

## 1. 概述

```
主版本 . 次要版本 . 补丁版本
```
一般情况下，次要版本号发生改变的话，表示程序有重大更新。

# 2. ~

Eg:
```
~2.3.4   2.3.4 <= version < 2.4.0
~2.3     2.3.0 <= version < 2.4.0
~2       2.0.0 <= version < 3.0.0
```

简要：
1. 补齐x.y.z格式；
2. 从右向左，找到第一个不为0；
3. 将不为零的+1，作为版本的上限；

# 3. ^

简要：
1. 补齐x.y.z格式；
2. 从左向右，找到第一个不为0；
3. 将不为零的+1，作为版本的上限；
4. 后续位改为0.
